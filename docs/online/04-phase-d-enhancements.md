# 阶段 D：增强功能

> 创建日期: 2026-04-19
> 状态: 待定（依赖阶段 C 完成）

## 目标

在核心多人功能就绪后，增加提升体验和可靠性的功能。

## 功能列表

### D1: AI 空位填充

**描述**: 当房间人数不足时，服务端自动添加 AI 玩家填入空座。真人加入时，AI 自动退出。

**规则**:
- 房间至少 2 名参与者才能开始（含 AI）
- 创建房间时可设置 `minPlayers` (默认 2)，AI 将自动补足到该数量
- 新真人加入时，最先加入的 AI 在当手结束后退出
- AI 难度随房间盲注级别自动调整

**实现要点**:
- 服务端运行 `decideAIAction` (复用 `src/ai/game-ai.js`)
- AI 行动时添加 500-2000ms 随机延迟
- AI 的昵称从预设池随机选取
- AI 的头像使用现有 Spy×Family 角色头像

---

### D2: 断线重连

**描述**: 玩家因网络问题断开后，可在一定时间内重新连接并恢复座位。

**规则**:
- 断线后服务端保留该座位 60 秒
- 保留期间，该玩家的行动由超时机制自动 check/fold
- 60 秒内玩家重连，恢复原座位并获取当前游戏状态
- 超过 60 秒，视为离开，座位释放

**实现要点**:
- 客户端: WebSocket `onclose` 触发指数退避重连
- 服务端: `GameSession` 维护 `disconnectedPlayers` Map
- 重连时发送 `RECONNECT` 消息，服务端验证身份后推送完整游戏状态

**WebSocket 重连流程**:
```
Client 断线
  → 立即显示 "正在重连..." 遮罩
  → 尝试 WebSocket 重连 (指数退避: 1s, 2s, 4s, 8s, 10s max)
  → 重连成功:
    → 发送 { type: 'RECONNECT', token, roomId }
    → 服务端验证 → 恢复 ws 绑定 → 推送 FULL_STATE
    → 客户端移除遮罩，继续游戏
  → 5 次重连失败:
    → 显示 "连接失败，返回大厅" 按钮
```

### D3: 观战模式

**描述**: 非参与者可以观看正在进行的游戏。观战者看到的信息受限（延迟 + 隐藏手牌对调整到与公平性对齐）。

**规则**:
- 观战者无法看到任何玩家的底牌（直到 showdown）
- 可以看到公共牌、底池、下注金额、玩家动作
- showdown 时可以看到所有亮出的牌
- 观战者的加入/离开对在局的玩家透明

---

### D4: 房间内聊天

**描述**: 房间内的实时文字聊天。

**规则**:
- 消息通过 WebSocket 广播给房间内所有人
- 基本的内容过滤 (敏感词)
- 消息速率限制 (每 2 秒最多 1 条)
- 可选的聊天开关

---

### D5: 排行榜

**描述**: 基于玩家总积分/胜率的全局排行。

**数据来源**:
- 每手牌结束后更新玩家统计
- 总手数、总胜局、总筹码变动、VPIP 等

**展示**:
- 大厅页面的排行榜面板
- Top 20 展示
- 个人排名查看

---

### D6: 自定义头像和昵称

**描述**: 玩家可以在个人设置中修改昵称，上传或选择头像。

**初期方案**:
- 昵称: 注册时设定，可在个人资料页修改
- 头像: 从预设头像列表中选择（暂不支持上传自定义图片）

## 功能优先级

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 (必须) | D1 AI 填充 | 没有足够真人时保证可玩性 |
| P0 (必须) | D2 断线重连 | 生产环境必备 |
| P1 (重要) | D5 排行榜 | 有账户系统则排行榜顺理成章 |
| P1 (重要) | D4 聊天 | 社交体验 |
| P2 (可选) | D3 观战 | 增强体验但非必须 |
| P2 (可选) | D6 自定义头像 | 个性化，非核心 |
