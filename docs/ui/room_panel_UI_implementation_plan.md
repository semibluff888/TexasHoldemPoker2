# 在线模式房间 UI 优化

## 问题分析

当前在线模式下，房间管理面板（Online Room Panel）被动态注入到左侧 `side-panel` 顶部，与 Action History 共用同一个 280px 宽、600px 高的容器：

### 核心问题

| 问题 | 描述 |
|------|------|
| **标题冲突** | 面板最顶部为 "ACTION HISTORY" 标题，但下方紧贴的是 Online Rooms 的 Refresh 按钮 和连接状态，语义混乱 |
| **空间挤压** | Room Panel 占据大量空间（状态条 + 创建表单 + 房间列表 + Leave 按钮），留给 Action History 的空间极少 |
| **缺少房间名输入** | 创建房间时没有房间名输入框（被挤掉了），只剩下人数选择和 Create 按钮 |
| **视觉层次混乱** | 两个面板用分隔线分开，但视觉上像是一个面板内的杂乱堆叠 |

## 设计方案

将左侧 `side-panel` 改造为**双标签页切换**布局：

```
┌─────────────────────────┐
│  [ 🏠 Room ] [ 📋 Log ] │  ← Tab 切换
├─────────────────────────┤
│                         │
│  (Active Tab Content)   │  ← 根据选中 tab 显示
│                         │
│                         │
└─────────────────────────┘
```

### Tab 1: Room（房间管理）

```
┌─────────────────────────┐
│  [ 🏠 Room ]  [ 📋 Log ]│
├─────────────────────────┤
│ 🟢 Connected as Guest 1 │  ← 连接状态徽章
├─────────────────────────┤
│ ┌─ Create Room ───────┐ │
│ │ Room Name  [______ ] │ │
│ │ Players    [5 ▾   ] │ │
│ │ [    CREATE ROOM   ] │ │
│ └─────────────────────┘ │
├─────────────────────────┤
│ ♠ Practice Table        │  ← 房间列表
│   2/5 · 10/20 · waiting │
│                  [ Join ]│
├─────────────────────────┤
│ [ 🚪 LEAVE ROOM       ] │  ← 底部固定
└─────────────────────────┘
```

### Tab 2: Log（操作历史）

保持现有 Action History 布局不变，包含手牌编号 + 日志列表 + 翻页导航。

### 自动切换逻辑

- 加入房间后，如果游戏开始（收到 `HAND_START`），自动切换到 Log tab
- 离开房间后，自动切换回 Room tab
- 用户可以随时手动切换

> [!IMPORTANT]
> 改动不影响 WebSocket 通信逻辑和游戏引擎。
